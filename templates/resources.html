Evaluation Metrics Guide

Classification Metrics

üéØ Accuracy

What it measures: Overall correctness of predictions

Formula: (True Positives + True Negatives) / Total Predictions

When to use: Balanced datasets, when all classes are equally important

‚ö†Ô∏è Caution: Can be misleading with imbalanced datasets
üîç Precision

What it measures: How many positive predictions were actually correct

Formula: True Positives / (True Positives + False Positives)

When to use: When false positives are costly (e.g., spam detection)

üé£ Recall (Sensitivity)

What it measures: How many actual positives were correctly identified

Formula: True Positives / (True Positives + False Negatives)

When to use: When false negatives are costly (e.g., medical diagnosis)

‚öñÔ∏è F1-Score

What it measures: Harmonic mean of precision and recall

Formula: 2 √ó (Precision √ó Recall) / (Precision + Recall)

When to use: When you need balance between precision and recall

Regression Metrics

üìä R¬≤ Score (R-squared)

What it measures: Proportion of variance explained by the model

Range: 0 to 1 (higher is better)

Interpretation: 0.8 means model explains 80% of the variance

üí° Tip: Good baseline metric for regression models
üìè Mean Absolute Error (MAE)

What it measures: Average absolute difference between predictions and actual values

Units: Same as your target variable

When to use: When you want easy-to-interpret error measurements

üìê Root Mean Square Error (RMSE)

What it measures: Square root of average squared differences

Characteristics: Penalizes large errors more than MAE

When to use: When large errors are particularly undesirable

üéØ Mean Squared Error (MSE)

What it measures: Average of squared differences

Characteristics: Always positive, heavily penalizes outliers

When to use: Mathematical optimization (used in training)

Best Practices & Tips

Model Development
Start Simple

Begin with simple models like Linear Regression or Logistic Regression to establish baselines.

Cross-Validation

Always use cross-validation to get reliable performance estimates and avoid overfitting.

Feature Importance

Understand which features drive your model's predictions for better interpretability.

Data Quality First

Spend time on data cleaning and preprocessing - it's often more important than algorithm choice.

Common Pitfalls
Data Leakage

Avoid using future information to predict the past. Check for features that shouldn't be available at prediction time.

Overfitting

When your model performs much better on training data than test data. Use cross-validation and regularization.

Ignoring Class Imbalance

When some classes are much more frequent than others. Use appropriate metrics and sampling techniques.

Not Validating Assumptions

Each algorithm has assumptions. Verify that your data meets these requirements for optimal performance.
